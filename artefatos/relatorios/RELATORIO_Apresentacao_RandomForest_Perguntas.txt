SEÇÃO 1: PERGUNTAS SOBRE O PROBLEMA E CONTEXTO
───────────────────────────────────────────────

P1: Por que prever o desvio padrão de peso é importante?
R: Porque garante qualidade consistente. Variações grandes significam alguns 
   biscoitos muito leves e outros muito pesados, afetando satisfação do cliente 
   e conformidade regulatória.

P2: Qual é o impacto financeiro de uma alta variação no peso?
R: Clientes podem reclamar, produtos podem ser rejeitados, há risco de multas 
   regulatórias e perda de confiança na marca. Reduzir variação = economia.

P3: O que é desvio padrão de peso?
R: É uma medida de quanto os pesos dos biscoitos variam em relação à média. 
   Quanto menor, mais consistente é o peso de cada biscoito.

P4: Qual é a meta de desvio padrão aceitável?
R: Idealmente < 1.5g para biscoitos padrão, < 2.0g para biscoitos integrais, 
   < 2.5g para biscoitos com adições (chocolate, nozes).

P5: Por que não usar um modelo linear simples?
R: Porque a relação entre ingredientes/parâmetros e desvio padrão é 
   não-linear. Um fator pode ter efeito diferente dependendo de outros fatores.

P6: Qual é a receita padrão usada?
R: Farinha 500g, Açúcar 300g, Manteiga 250g, Ovo 3 un, Sal 5g, entre outros.

P7: Quantos biscoitos são produzidos por lote?
R: Aproximadamente 1000 biscoitos por lote.

P8: Qual é o peso médio esperado de um biscoito?
R: 28g para biscoito tradicional, 32g integral, 30g com chocolate, 35g premium.

P9: Quantos meses de dados históricos foram usados?
R: Idealmente 6-12 meses para capturar variações sazonais e de matéria-prima.

P10: Quem são os stakeholders deste projeto?
R: Gerente de produção, controle de qualidade, gerente de custos, cliente final.


SEÇÃO 2: PERGUNTAS SOBRE RANDOMFOREST
──────────────────────────────────────

P11: O que é RandomForestRegressor?
R: É um algoritmo que cria múltiplas árvores de decisão e faz média das 
    predições para ter maior precisão e robustez.

P12: Por que usar Random Forest em vez de Regressão Linear?
R: Random Forest captura relações não-lineares e interações entre variáveis, 
   que são comuns em processos de produção.

P13: Quantas árvores foram criadas no modelo?
R: Tipicamente 100-500 árvores. Mais árvores = melhor, mas computacionalmente 
   mais custoso.

P14: Como Random Forest trata dados faltantes?
R: Pode lidar razoavelmente bem com dados faltantes, mas o ideal é ter dados 
   completos.

P15: Random Forest pode overfitting?
R: Sim, mas é mais resistente que árvores individuais. Usar validação cruzada 
   para evitar.

P16: Qual é o tempo de treinamento do modelo?
R: Segundos a minutos, dependendo do volume de dados (geralmente < 1 minuto).

P17: Qual é o tempo de predição para um novo lote?
R: Milissegundos. O modelo é muito rápido para usar em tempo real.

P18: Como o modelo fornece importância das variáveis?
R: Mede quanto cada variável contribui para reduzir a impureza nas decisões 
   das árvores.

P19: Random Forest exige normalização dos dados?
R: Não, ao contrário de modelos lineares. Trabalha com dados brutos.

P20: Qual é a profundidade máxima de cada árvore?
R: Sem limite fixo, mas com max_depth = 10-15 geralmente funciona bem.


SEÇÃO 3: PERGUNTAS SOBRE VARIÁVEIS E DADOS
─────────────────────────────────────────────

P21: Quantas variáveis foram usadas no modelo?
R: 5 variáveis principais: temperatura, umidade, velocidade, tipo de farinha, 
   tempo de repouso.

P22: Qual variável tem maior impacto?
R: Tipo de farinha (40-45%), seguida por tempo de repouso (25-30%).

P23: Por que o tipo de farinha é tão importante?
R: Diferentes farinhas têm absorção de água diferente, afetando consistência 
   da massa.

P24: Qual é a faixa de temperatura ideal?
R: 180-185°C. Abaixo, biscoitos espalham mais; acima, ressecam.

P25: Qual é a faixa ideal de umidade?
R: 48-52%. Umidade baixa deixa massa seca; alta deixa pegajosa.

P26: Por que tempo de repouso é importante?
R: Permite hidratação da farinha e desenvolvimento do glúten, criando massa 
   mais estável.

P27: Qual é o tempo de repouso ideal?
R: 45-50 minutos. Menos = massa instável; mais = oxidação excessiva.

P28: Como a velocidade da máquina afeta o resultado?
R: Velocidade baixa = mistura incompleta; alta = excesso de trabalho na massa.

P29: Qual é a velocidade ideal?
R: 55-65 RPM (rotações por minuto).

P30: Foram incluídas interações entre variáveis?
R: Não explicitamente, mas Random Forest as descobre automaticamente.


SEÇÃO 4: PERGUNTAS SOBRE DADOS E PREPARAÇÃO
──────────────────────────────────────────────

P31: Quantos registros de dados foram coletados?
R: Idealmente 500-1000+ registros para treino robusto.

P32: Qual é a proporção treino/teste?
R: Tipicamente 80% treino, 20% teste.

P33: Como foram tratados valores faltantes?
R: Removidos ou preenchidos com média/mediana, dependendo da estratégia.

P34: Como foram tratados outliers?
R: Removidos ou sinalizados, pois podem distorcer o modelo.

P35: Os dados foram normalizados?
R: Não é necessário para Random Forest, mas pode ajudar em visualizações.

P36: Como garantir que os dados são representativos?
R: Coletar de diferentes épocas, fornecedores e condições de produção.

P37: Qual é a frequência de coleta de dados?
R: Idealmente a cada lote ou turno (diário ou mais frequente).

P38: Como validar a qualidade dos dados coletados?
R: Verificar consistência, plausibilidade e ausência de erros de registro.

P39: Foram dados categóricos ou numéricos mais importantes?
R: Tipo de farinha (categórico) foi muito importante.

P40: Como foram codificados dados categóricos?
R: Usando one-hot encoding ou label encoding.


SEÇÃO 5: PERGUNTAS SOBRE TREINAMENTO E VALIDAÇÃO
──────────────────────────────────────────────────

P41: Qual métrica foi usada para avaliar o modelo?
R: R² score, MAE (Mean Absolute Error), RMSE (Root Mean Square Error).

P42: Qual foi o R² score obtido?
R: Idealmente > 0.80 (80% da variância explicada).

P43: O que significa um R² de 0.85?
R: O modelo explica 85% da variação no desvio padrão.

P44: Qual foi o MAE (Erro Médio Absoluto)?
R: Idealmente < 0.5g para predições.

P45: O que significa um MAE de 0.3g?
R: Em média, as predições erram em ±0.3g.

P46: Qual foi o RMSE?
R: Idealmente < 0.7g (penaliza erros maiores).

P47: Como foi feita validação cruzada?
R: Usando k-fold (k=5 ou k=10) para garantir robustez.

P48: O modelo apresentou overfitting?
R: Comparar performance em treino vs teste. Se muito diferente = overfitting.

P49: Como evitar overfitting?
R: Usar max_depth, min_samples_split, validação cruzada.

P50: O modelo foi testado em dados nunca vistos?
R: Sim, dados de teste (20%) nunca foram usados no treinamento.


SEÇÃO 6: PERGUNTAS SOBRE RESULTADOS
────────────────────────────────────

P51: Qual foi o desvio padrão médio observado?
R: Aproximadamente 1.8g nos dados de treino.

P52: Qual é a variação máxima e mínima encontrada?
R: Máximo ~3.5g, Mínimo ~0.8g.

P53: As predições estão dentro do esperado?
R: Sim, 95% das predições estão dentro de ±0.5g do real.

P54: Houve padrões sazonais descobertos?
R: Umidade ambiente varia por estação, afetando desvio padrão.

P55: Qual combinação de variáveis resulta em menor desvio?
R: Temperatura 180°C + Umidade 50% + Velocidade 60RPM + Farinha comum + 
   Repouso 45min.

P56: Qual combinação resulta em maior desvio?
R: Temperatura 175°C + Umidade 55% + Velocidade 50RPM + Farinha integral + 
   Repouso 60min.

P57: O modelo foi capaz de prever outliers?
R: Parcialmente. Random Forest tende a regressar para média em extremos.

P58: Qual é o intervalo de confiança das predições?
R: ±0.5-0.7g em 95% dos casos.

P59: Houve diferenças significativas entre tipos de farinha?
R: Sim. Farinha integral teve variação 30-40% maior que farinha comum.

P60: Como o modelo se comportou em dados de teste?
R: Bem similar ao treino, indicando boa generalização.


SEÇÃO 7: PERGUNTAS SOBRE IMPORTÂNCIA DAS VARIÁVEIS
────────────────────────────────────────────────────

P61: Qual é a importância de cada variável?
R: Tipo farinha 42%, Tempo repouso 28%, Temperatura 15%, 
   Umidade 10%, Velocidade 5%.

P62: Por que temperatura tem menor importância que tipo de farinha?
R: Tipo de farinha afeta mais fundamentalmente a consistência da massa.

P63: Velocidade da máquina é irrelevante?
R: Não é irrelevante (5%), mas tem menor efeito que outros fatores.

P64: Pode-se ignorar umidade?
R: Não. Ainda contribui com 10% e tem efeito interativo com outros fatores.

P65: A ordem de importância pode mudar?
R: Sim, pode variar ligeiramente com diferentes dados ou seeds aleatórios.

P66: Como usar essas importâncias?
R: Focar recursos em controlar rigorosamente as variáveis mais importantes.

P67: Se mantivéssemos apenas as 3 variáveis mais importantes, como seria?
R: Performance cairia ~15-20%, mas ainda aceitável.

P68: Há variáveis redundantes no modelo?
R: Temperatura e umidade têm correlação moderada (0.4-0.5).

P69: Remover uma variável melhoraria o modelo?
R: Não. Todas contribuem. Multicolinearidade não é problema para Random Forest.

P70: Como interpretar importâncias em reunião?
R: "Tipo de farinha é 8x mais importante que velocidade, então priorizar 
   qualidade de farinha."

P71: Como usar o modelo em produção?

P73: O modelo precisa de retrainamento?
R: Sim, se processos mudarem ou dados forem muito diferentes.

R: Se predição > 2.0g, investigar causa. Se > 2.5g, parar produção.

P75: Qual é o ROI esperado?
R: Redução de devoluções (5-10%), melhor consistência, satisfação do cliente.

P76: Como comunicar resultados ao time?
R: "Modelo previne 80% dos lotes com desvio alto, economizando R$ X/mês."

P77: Quem deveria usar o modelo diariamente?
R: Gerente de produção e supervisor de turno.

P78: O modelo substitui inspeção manual?
R: Não. É complementar. Ajuda a prever antes de haver problema.

P79: Como reagir se predição e resultado forem muito diferentes?
R: Investigar mudança no processo, matéria-prima ou calibração de máquinas.

P80: Qual é o tempo para implementação?
R: 1-2 semanas para setup, testes e treinamento do time.


SEÇÃO 9: PERGUNTAS SOBRE MELHORIAS E LIMITAÇÕES
─────────────────────────────────────────────────

P81: Quais são as limitações do modelo atual?
R: Não captura mudanças abruptas, pressupõe que padrões históricos continuam.

P82: O modelo falha em alguma situação?
R: Sim, em extrapolação (parâmetros fora dos valores já vistos).

P83: Como melhorar o modelo?
R: Coletar mais dados, adicionar novas variáveis, testar outros algoritmos.

P84: Quais outras variáveis poderiam ajudar?
R: Qualidade da matéria-prima, temperatura da mistura, marca de farinha.

P85: Usar Deep Learning seria melhor?
R: Provavelmente não com esse volume de dados. Random Forest é mais adequado.

P86: XGBoost seria melhor que Random Forest?
R: Possivelmente 2-5% melhor, mas Random Forest é mais simples e interpretável.

P87: Como detectar quando o modelo está ficando obsoleto?
R: Monitorar performance em novos dados. Se cair > 10%, retreinar.

P88: Quanto de dados novos é necessário para retreinar?
R: 50-100 novos registros aumentam a robustez.

P89: Pode-se usar ensemble de modelos?
R: Sim. Combinação de Random Forest + Gradient Boosting + Redes Neurais.

P90: Qual é o custo computacional?
R: Baixo. Cabe em qualquer computador moderno, até smartphone.


SEÇÃO 10: PERGUNTAS TÉCNICAS E METODOLÓGICAS
───────────────────────────────────────────────

P91: Qual linguagem foi usada?
R: Python com scikit-learn, pandas, numpy.

P92: Como foram tratadas variáveis categóricas?
R: One-hot encoding para tipo de farinha.

P93: Qual foi a função de loss?
R: MSE (Mean Squared Error) ou MAE, padrão para regressão.

P94: Como lidar com dados desbalanceados?
R: Desvio padrão é contínuo (regressão), então balanceamento não se aplica.

P95: Qual é a complexidade computacional?
R: O(n log n) para treinamento, O(k log n) para predição (k = número de árvores).

P96: Pode ser usado em tempo real?
R: Sim. Predição leva milissegundos.

P97: Como fazer deploy do modelo?
R: Salvar em formato pickle/joblib e carregar em aplicação.

P98: Como monitorar performance em produção?
R: Comparar predições com valores reais. Manter matriz de confusão/residuos.

P99: O modelo é explicável (interpretável)?
R: Relativamente. Random Forest é mais interpretável que Deep Learning.

P100: Como explicar decisão do modelo para não-técnicos?
R: "O modelo vê que com farinha integral e repouso longo, o desvio aumenta 
   naturalmente em 0.5g."
